Dealer-Focused MVP

[App Name] Show Dealer Tool

This version is intentionally narrower than the broader event concept. It is built for the semi-pro show dealer running 1 to 4 tables who needs to price bundles fast, close deals, and reconcile with eBay later.

  • Single workflow: bundle scan -> aggregate pricing -> deal log
  • Dark mode and oversized actions for harsh convention lighting
  • Offline-aware patterns and minimal text input
  • No browsing, no portfolio, no social clutter
Review Lens

What to pressure-test

  • Does the scanner screen feel like a faster replacement for CollX + calculator + notes?
  • Are aggregate low / mid / high values visible enough during the deal?
  • Does the suggested bundle price feel useful without over-automating judgment?
  • Is Deal Log lean enough to be operational, not “inventory lite”?
  • Does eBay sync feel like relief for a painful workflow?
User

Semi-Pro Show Dealer

Regularly attends shows, has active eBay listings, and needs a floor tool that minimizes typing and context switching.

Core Promise

Price Bundles While Talking

Scan cards into a running bundle, see the range and suggested quote, then close the deal without opening other apps.

Not Included

No Marketplace, No Portfolio

V1 excludes collection tracking, social features, maps, and discovery. This is a transaction tool.

Design Notes

Decisions Worth Locking Before PRDs

Scanner

Suggested Bundle Price

The wireframes assume a configurable dealer margin that converts `mid` value into a suggested close price. We should define whether that margin is global, show-specific, or editable per bundle.

Trust

Pricing Source Visibility

`eBay last 30 days` is shown persistently on the scanner because this number needs to be defensible during a live deal.

Operations

Offline Behavior

The screens assume cached comp data and graceful offline states. The PRD should define exactly what still works with degraded WiFi.

Scope

Keep Deal Log Final

Preventing post-close edits keeps the log operational and reduces the risk of sliding into full inventory management too early.
