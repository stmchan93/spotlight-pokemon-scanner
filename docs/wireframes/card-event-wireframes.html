Low-Fi Wireframe Deck

Spotlight Event Card App

This deck is intentionally low fidelity. It is for testing flow, hierarchy, trust signals, and the event-floor wedge before formal PRDs or visual design.

  • Primary wedge: fast, trustworthy deal quoting for live events
  • Shared core for buyers and vendors, with lightweight mode shifts
  • Core object: Deal Stack, not a generic portfolio or marketplace list
Review Filter

What to react to

  • Is the first value screen fast and trustworthy enough?
  • Does Deal Stack feel right for both buyers and vendors?
  • What must stay above the fold during negotiation?
  • Should buyer and vendor share the same screen with different emphasis?
  • Is the credibility layer clear without being annoying?
Wedge

Quote Faster Than Search

The product should feel better than opening three marketplace apps, searching card by card, then doing mental math in front of another person.

Core Trust

Evidence With Every Number

Every price summary should explain freshness, comp count, and whether the number is exact-card data or inferred.

Scope

Start With Stacks, Not Inventory

Saved stacks can later evolve into trays, folders, and inventory. Do not make setup heavier than the event use case needs.

Decision Notes

What We Should Pressure-Test Next

Naming

Deal Stack vs Vendor Tray

Buyer-facing language may prefer Deal Stack. Vendor-facing language may prefer Tray or Quote. We should test whether one object with mode-specific labels feels cleaner.

Trust

How Visible Should Credibility Be?

The trust module is essential, but it can overwhelm small screens. We need to decide whether it is always expanded or collapses after the first view.

Vendor Speed

Continuous Scan

Vendor mode may need a camera sheet that stays open while totals update in the background. This deck shows the simpler version first.

Scope

Inventory Later

If these screens feel strong, PRDs should lock v1 around scan, search, result, stack, and quote only. Full inventory can remain a later phase.
