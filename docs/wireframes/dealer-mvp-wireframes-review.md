# Dealer MVP Wireframe Review

Primary artifact:

- [dealer-mvp-wireframes.html](/Users/stephenchan/Code/spotlight/docs/wireframes/dealer-mvp-wireframes.html)

This deck is based on the tighter MVP:

- Target user: semi-pro show dealer
- Core workflow: `bundle scan -> aggregate pricing -> deal log`
- Navigation: `Scanner`, `Want List`, `Deal Log`, `Profile`

Review these decisions first:

1. Is the `Scanner` screen strong enough to replace `CollX + notes app + calculator`?
2. Are low / mid / high totals and the suggested bundle price visible enough during the deal?
3. Does `Deal Log` feel like a final floor ledger instead of accidental inventory software?
4. Does the `eBay Sync` setup / reconcile flow address the most painful operational gap?
5. Is anything here still too broad for v1?

Best next step after your feedback:

- Lock the screen set and convert it into a short MVP PRD with user stories, acceptance criteria, offline assumptions, and phase boundaries.
